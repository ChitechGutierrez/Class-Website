Portfolio Website Project

Juniors/Seniors: MMAII 2015-16

Updated on March 7th, 2016

For this project students created portfolio websites where their work to be kept for the entire year. Every student does multiple project at a time each year, and their work should live somewhere after they are done with it. Students designed these sites themselves, building from HTML and CSS code, with the help of the Bootstrap and jQuery libraries to make it easier. Honors students were not allowed to use a Bootstrap skeleton, but they could use different parts of the library. Websites will be created on c9.io, and are hosted as a Github website.

This project took far too long, as the students had to complete a CodeAvengers course before starting on the project. They ended up getting a little burnt out. We ended up running into winter break, which gave them a break. This project also ran up against exhibition, and would not have been a terrribly exchiting project to exhibit, we took a break and did the Pixelated Life Project right after winter break. We then came back to the Portfolio Project, with new content to put in the sites. After some review time, the students worked, and again, struggled. In the end they learned things through the struggle, but a bit more structure and a bit less freedom for this project would have been better. I didn't have them use templates later in the process. Feedback that I received from the students suggested having them start with templates instead.

    Essential Questions:

  • What is high quality work?
  • How do I present myself and my work to the world?
  • Is it the message I want to send? Why do I send that?

    Learning Goals:

  • Using GitHub
  • HTML5 and CSS3
  • Bootstrap and jQuery - Using programming libraries
  • What is a Library? Why are they so useful.
  • What is a class/id? How do you use them.
  • How to do a wireframe/prototype.
  • How to properly organize and HTML and CSS document.
